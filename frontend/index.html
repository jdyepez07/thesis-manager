How it works

Three steps from repository to running container.

link
STEP 01

Paste the GitHub repository URL

Students submit their projects as public GitHub repositories containing a Dockerfile or docker-compose.yml. The reviewer pastes the URL into the deploy form.

tune
STEP 02

Choose a name and port

Set a project identifier and the host port where the app will be accessible. The platform clones the repository, patches the port configuration automatically, and strips incompatible volume mounts from the student's compose file.

check_circle
STEP 03

Project is live — open it directly

The backend builds and starts the container in the background. Once running, the dashboard shows a live status badge and an Open button that takes you directly to the deployed project.

What the platform handles

Everything that would otherwise be done manually.

cloud_download

Automatic repository cloning

Runs git clone on the provided URL and stores the project locally. Re-deploying the same project skips the clone step.

swap_horiz

Port rewriting

Patches the external port in docker-compose.yml before deploying, supporting string, integer and dictionary port formats.

dns

Compose and Dockerfile support

Detects whether the project uses docker-compose.yml or a plain Dockerfile and applies the correct build strategy automatically.

monitoring

Live container status

The dashboard polls the backend every 5 seconds after a deploy and updates the status badge when the container transitions to Up.

stop_circle

One-click stop and remove

Each running container has a Stop button that calls docker stop and docker rm in sequence, freeing the port immediately.

open_in_new

Direct project access

Once a container is running, the dashboard shows an Open button that resolves the host port from the container metadata and opens the project in a new tab.

Student requirements

What a thesis project must include to be deployable.

school Checklist for student repositories
check_circle

Public GitHub repository

The platform clones via HTTPS. Private repositories are not supported.

check_circle

Dockerfile or docker-compose.yml at the root

The platform detects which one is present and applies the corresponding build strategy.

check_circle

Application exposed on port 5006 (Dockerfile only)

Projects using a plain Dockerfile must run their server on internal port 5006. Projects using docker-compose.yml can declare any port.

info

No host-specific volume mounts

The platform strips all volume definitions from docker-compose.yml before deploying. Avoid depending on local paths being mounted.
